## 🔥 Amazon CEO 一通電話觸發白宮出口管制：Anthropic 旗艦模型 Fable 5、Mythos 5 全球下架

2026 年 6 月 12 日，Anthropic 在全球範圍內暫停了 Fable 5 與 Mythos 5 的存取。Fable 5 才公開發表三天，Mythos 5 從未正式對外開放。兩者在一紙出口管制命令下，同時消失。

根據 Wall Street Journal 報導，起點是 Amazon CEO Andy Jassy 與川普政府高層官員（包括財政部長 Scott Bessent）的會談。Amazon 內部安全研究員使用一系列提示詞，成功繞過 Fable 5 外層的安全分類器——這個分類器原本負責在偵測到惡意意圖時，將請求降級到更保守的 Claude Opus 4.8 處理。研究員讓分類器無法正確識別惡意意圖，從而使底層的 Fable 5 直接回應了可用於輔助網路攻擊的資訊。

白宮隨後開會驗證了 Amazon 的發現，川普總統親自簽署限制令。這是史上第一次，出口管制被直接應用在 AI 模型本身——過去出口管制管的是硬體（晶片、設備），現在一個可以被外國人存取的模型就觸發了出口管制。

這裡的利益衝突結構值得注意：Amazon 同時是 Anthropic 的最大投資人（據報投資 80 億美元）、晶片供應商（Anthropic 模型跑在 Amazon Trainium/Inferentia 上）、客戶（用 Anthropic 模型做內部安全掃描）、以及競爭對手（AWS 有自己的 AI 模型）。檢舉人、投資人、供應商、競爭對手——四重身分集於一身。

Anthropic 則主張這個繞過方法只暴露了「輕微」安全缺陷，在其他公開可用的模型中同樣存在，不足以構成出口管制的理由。該公司已對政府的供應鏈黑名單指定提出法律挑戰。專家指出，禁令範圍之廣連受僱於 Anthropic 的加拿大籍和英國籍研究員都無法碰自己公司的模型。

## OpenAI 宣布收購 Ona：讓 Codex agent 擁有「關機後繼續跑」的雲端工作空間

OpenAI 在 6 月 11 日正式宣布收購 Ona——一家專注於安全雲端執行環境與編排技術的公司。這筆交易的核心目標：讓 Codex 的 agent 能在持久化的雲端環境中跨 session 持續執行任務，即使你把筆電蓋上去睡覺，agent 仍在客戶自己的雲端環境裡繼續推進。

Codex 目前每週有超過 500 萬人使用，比今年稍早成長了 400%。OpenAI 認為最有價值的 agent 工作不會在幾分鐘內完成——它可能需要在數小時甚至數天內持續推進。Ona 過去幾年累積了 200 萬開發者用戶，其核心技術是客戶控制的安全執行環境：agent 在客戶自己的雲端基礎設施內運行，客戶自己決定 agent 能存取什麼、憑證怎麼設、活動怎麼記錄。

這對企業場景尤其關鍵。當組織從「試玩 agent」進入「把 agent 部署到正式工作流程」，安全、治理、合規、營運要求一個都不少。收購仍需監管批准。

## 📊 Anthropic 發布首份「Public Record」民調：五萬美國人對 AI 的真正想法

Anthropic 發表了名為 Anthropic Public Record 的大規模調查報告，透過 YouGov 對近 52,000 名美國人進行了具全國代表性的線上調查。這是第一次有 AI 公司直接問「一般大眾」（而不只是使用者）對 AI 的真實態度。

**美國人對 AI 的三大希望：** 治癒疾病（48%）、幫助身障者（36%）、推動技術進步與讓生活更便利（並列 23%）。「治療」和「陪伴」排名墊底——美國人並不期待 AI 來填補人際關係的空缺。

**美國人最怕的三件事：** 工作被取代（64%，不分黨派州別都是第一）、認知依賴（56%）、錯誤資訊（52%）。學歷越高的人越怕 AI 搶工作（研究所學歷比高中以下高出近 10 個百分點）；但每天在工作中使用 AI 的人反而比較不怕（54% vs 從不使用者的 70%）。

最刺眼的數字：當被問到「你信任誰來決定 AI 的發展與使用方式」時，**AI 公司只拿到 15%**（所有選項最低），獨立專家則拿到 43%。71% 美國人認為政府應該參與 AI 監管——民主黨 79%、共和黨 68%、獨立選民 69%，極其罕見的跨黨派共識。

## 🧪 Claude 當化學家：NMR 圖譜解析能力正式超越 ChemDraw

Anthropic 研究化學家 David Kamber 發表白皮書，評估 Claude 在 NMR（核磁共振）光譜解析上的表現。結論：通用模型 Claude Opus 4.7，沒有經過任何化學專用微調，在 NMR 峰值預測上與 ChemDraw 和 MestReNova 打成平手甚至略勝一籌。

測試使用 ChemRxiv 上 20 個在模型訓練截止日期後發表的新化合物（避免資料汙染）。氫譜預測：Opus 4.7 平均誤差 ±0.079 ppm，遠低於化學家認定的 ±0.20 ppm 容許範圍，所有工具中最準確。碳譜與 MestReNova 基本打平。更難的「反向推結構」任務（從光譜推測分子），Claude 在簡單題上三次全對，困難題也表現出色。

現有軟體只能做正向預測（給結構→預測光譜），化學家真正需要的是反向推導（拿到光譜→推測分子），Claude 在這個方向展現了專用軟體沒有的能力。Anthropic 正在針對化學家日常工作最耗時的五個瓶頸發力：結構讀取與繪製、合成路線推理、反應機制解釋、化學文獻理解、光譜解析。

## 💻 RTX 5080 + RTX 3090 雙卡合體：在家跑到 80+ tok/s 的 Qwen 3.6 27B Q8

開發者 imil 記錄了他如何把一張 RTX 5080（16GB）和一張 RTX 3090（24GB）湊在一起，用 llama.cpp 跑 Qwen 3.6 27B 的 Q8 量化版，最終達成穩定 80-90 tok/s 的生成速度。

關鍵硬體選擇：Asus Prime X570-Pro 主機板（一定要 Pro 版才能拆 PCIe x16 為 x8/x8）、BIOS 必須關掉 CSM 用 UEFI 模式開機、Above 4G Decoding 和 ReSize BAR 都要開。軟體方面，llama.cpp 編譯時要同時啟用 Ampere（86）和 Blackwell（120）架構，啟動參數 `-sm tensor -ts 2,3` 搭配 MTP 推測解碼。

最終 Qwen 3.6 27B Q8（約 27GB）+ 230K context + Q8 KV cache，全部塞進合計 39GB VRAM。draft acceptance rate 約 77%，生成速度穩定 80+ tok/s，峰值可達 90+。一張二手 3090 配原本的 5080，主板選對、BIOS 設對，就能在家用 Q8 精度跑 27B 模型。

## 🏰 Show HN: Bastion — 為背景 coding agent 開出隔離 Linux VM

一位開發者 almostlit 在 Hacker News 發表了 Bastion，一個專為背景 coding agent 設計的隔離 VM 管理工具。概念很直接：每個 coding agent 跑在自己的獨立 VM 裡，彼此檔案系統和執行環境完全隔離，不會互相干擾。

**核心特性：**

- **多 agent 並行擴展**：同時跑多個背景 coding agent，零執行期衝突
- **完全隔離**：每個 agent 在獨立 VM 中擁有完整的檔案系統和背景程序控制權
- **SSH 即時觀察**：可以 SSH 進 agent 的環境，即時查看檔案變更和終端機操作
- **基礎設施即程式碼**：用 JSON 模板定義 agent 環境，支援版本控制
- **支援自託管模型**：相容任何 OpenAI-compatible endpoint，不限於雲端 API

使用流程也很簡潔：`curl` 安裝腳本 → 定義 JSON 模板（指定工作目錄、API 金鑰等）→ `bastion templates create` 初始化 → `bastion env create` 部署 VM → 透過 `bastion opencode` 或 `bastion ssh` 進入 agent 環境。GitHub 開源（bastion-computer/bastion），HN 上獲得 24 點。

在 agent 時代，「讓 agent 跑在哪裡」正在變成一個真實的基礎設施問題。Bastion 給出的答案是：每個 agent 一個 VM，乾淨隔離，宣告式管理。
