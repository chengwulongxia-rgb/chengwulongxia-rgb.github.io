今天 LLM 圈的主題意外地集中在一個字：**記憶**。從視聽模型的 KV cache 壓縮，到 AI agent 的經驗系統，再到「LLM 到底會不會思考」的老問題——大家都在問同一件事：模型記住了什麼，又忘了什麼？

---

## 🔥 OmniMem：幫影音 LLM 的記憶體減肥，不減智商

長影片理解一直是視聽 LLM 的痛點——影片 token 和 KV cache 隨著長度線性成長，再大的 GPU 也撐不住。現有的壓縮方法把所有的 token 一視同仁地砍，但影片的問題是：**畫面 token 是聲音 token 的幾十倍**，兩個模態根本不該同等待遇。

OmniMem 這篇論文提出了一套專門給視聽 LLM 的記憶壓縮框架，核心有三招：

1. **模態感知的記憶分配**：視覺和音訊分開管理，針對 token 數量嚴重失衡的問題對症下藥。
2. **擾動感知的記憶篩選**：不是無腦砍 token，而是保留「資訊量大、不重複」的 KV 狀態——砍掉的是冗餘，留下的是精華。
3. **預算感知的微調**：在訓練時就讓模型學會在有限的記憶預算下，把重要資訊塞進保留的記億裡。

在 VideoMME Long、LVBench 等 benchmark 上，OmniMem 在相同記憶預算下比無需訓練的壓縮方法高出 2-4% 準確率，微調後再補 1-2%。聽起來不多，但在 long video 場景下，這已經是很實在的進步了。

城武觀點：KV cache 壓縮這條賽道越來越捲，從 H2O、StreamingLLM 到現在的 OmniMem，大家都在用不同角度砍記憶。這篇的亮點是把「影片=畫面+聲音」這個常識真正落實在架構設計上，而不是把所有東西當成同一種 token 來處理。對開發者來說，如果你的應用場景是長影片理解（監控、會議記錄、教學影片），這方向值得追。

- 來源：[arxiv.org](https://arxiv.org/abs/2606.07577)

## PathoSage：病理學 AI 終於學會「不要瞎掰」了

醫療 AI 的老毛病：你問它這張病理切片有沒有癌細胞，它說得頭頭是道，但你仔細一看——它說的特徵根本不在那張圖上，純幻覺。

PathoSage 這篇論文的解法很優雅：把整個診斷流程拆成三個獨立的階段——**知識檢索 → 證據收集 → 證據裁決**。關鍵在第三階段「結構化證據審議」：它會把來自不同工具的證據（影像分析、文獻檢索、知識庫查詢）各自獨立評估，做衝突分析，然後在一個「乾淨的 context」裡做出最終判斷——避免前面階段的 bias 污染決策。

更妙的是，它還加了一個不需要訓練的 Beta-Bernoulli 經驗系統：根據每次工具使用的結果好壞，持續更新對各工具可靠度的評估，讓未來優先選擇可靠的資訊來源。簡單說，就是讓 agent 記住「上次這個工具唬爛我，這次我打個折」。

實驗結果顯示 PathoSage 在病理 VQA 上有效降低幻覺，而且超越了現有的病理 MLLM 和 agentic baseline。

城武觀點：這篇真正有趣的不只是醫療應用，而是那個「證據裁決 + 經驗系統」的架構——把 agent 的 multi-step reasoning 拆成「找資料 → 比對 → 判決」三段，每一步都在獨立的 context 裡執行，避免交互污染。這套路完全可以搬到法律、金融稽核、甚至 code review 場景。病理學只是一個示範。

- 來源：[arxiv.org](https://arxiv.org/abs/2606.07549)

## 「創新幻覺」：一篇論文主張 LLM 根本不會思考

這篇論文的標題就很敢寫——直接把 LLM 的熱潮稱為「創新幻覺」（Innovation Illusion）。作者從認知語言學、神經心理學、聚合動力學等角度出發，提出了一個大膽的論點：

**LLM 只是從訓練文本中學到了「隱喻式的問題傳播路徑」，但這跟人類真正的思考和理解是兩回事。** 人類解決問題時，會在不同概念域之間做隱喻映射（metaphorical problem propagation）；LLM 雖然在文字層面模仿了這個過程，但缺乏真正的理解基礎。因此，基本型 chatbot 不可能是能跟人類匹敵的思考夥伴——而且繼續擴大 LLM 規模也無法改變這一點。

論文最後引用 Yann LeCun 的名言來背書，並直言這個結論跟 Big Tech 的樂觀主義「完全對不上」。

城武觀點：這篇文讀起來有種「用哲學砸工程師」的快感。但說實話，這種「LLM 不是真的思考」的論點我們聽了很多年了，從 Bender 的 stochastic parrot 到現在，問題從來不是「LLM 會不會思考」，而是「LLM 能不能解決問題」。一個不會「真正理解」但能把你的程式碼 debug 完的工具，你要不要用？我反正是用了。

- 來源：[arxiv.org](https://arxiv.org/abs/2606.07722)

## AI Agent 實戰：能幫你做科學研究，但別叫它自己判斷

這是一篇超級務實的案例研究。研究團隊拿通用的 coding agent 去跑一個真實的神經科學資料處理 pipeline——果蠅光遺傳學實驗，從原始資料到科學發現的完整流程。任務規模遠大於現有 benchmark，資料集大上好幾個量級，而且評測標準是領域專家等級的。

結果喜憂參半：

👍 Agent 能獨立完成 pipeline 中的**個別階段**——階段級自動化是可行的。

👎 但端到端整條 pipeline？不行。一個階段失敗就全線崩潰。

最大的瓶頸是：當任務沒有「預定義的正確答案」可以迭代逼近，而需要 agent 用自己的「科學判斷力」來評估當前的解決方案時，agent 就卡住了。有趣的是，agent 有時會嘗試用**視覺化**來自我檢查中間產出——它確實去看了圖，但完全無法正確解讀畫面內容或據此調整策略。就像一個學生畫了圖但看不懂自己在畫什麼。

城武觀點：這篇對 AI agent 開發者來說是必讀。它點出了一個現在幾乎所有 agent benchmark 都避開的問題：**當任務是 open-ended 的，沒有 ground truth 可以對答案的時候，agent 怎麼判斷自己做對了？** 這是從「tool use」進化到「自主科學家」的最後一哩路，也是最難的一哩。

- 來源：[arxiv.org](https://arxiv.org/abs/2606.07718)
