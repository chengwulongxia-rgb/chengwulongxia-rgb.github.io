## 🔥 Claude Tag：Anthropic 把 Claude 變成你的 Slack 同事

Anthropic 今天推出 Claude Tag——一個讓 Claude 以 @Claude 身份加入 Slack 頻道的新產品。跟過去「開一個對話視窗、丟 prompt、等回應」的互動模式不同，Claude Tag 的設計邏輯是把 Claude 定位成一個會記住上下文、會主動回報、可以在背景非同步工作的「團隊成員」。

幾個關鍵機制：

**多人共用同一個 Claude。** 在同一個 Slack 頻道裡，所有人都跟同一個 Claude 互動。任何人可以看到 Claude 正在處理什麼，也可以接續上一個人的對話。Anthropic 的說法是「這跟和同事協作比較像，而不是開一個單人聊天視窗」。

**Claude 會「學習」。** 只要被授權，Claude 會讀取頻道內的歷史訊息、甚至跨頻道和資料來源自動建立上下文。Anthropic 強調 Claude 不會讀取私人頻道。但「學習」這個詞本身值得停下來想——Claude 沒有在學習，它在讀你的對話紀錄，把內容放進 context window。把這叫「學習」是一個敘事選擇。

**Claude 會「主動出擊」。** 如果開啟 ambient 模式，Claude 會主動推送它認為你該知道的資訊——跨頻道標記相關內容、追蹤未解決的討論串。換句話說：Claude Tag 不只在等你 @它，它在監看你頻道裡的對話。

**非同步工作。** 你可以丟一個任務給 Claude，然後去做別的事。Claude 甚至可以自己排程——設定它在未來幾小時或幾天內自主完成一個專案。Anthropic 表示內部已經大量採用這種模式：「我們現在花更多時間在平行委派任務給多個 Claude。」

Anthropic 給了一個數字：內部產品團隊 65% 的程式碼是由 Claude Tag 產生的。這個數字沒有附帶任何審查比例、修改比例、或是「65% 是指 commit 數量還是行數」的定義。但它被放在產品發布文宣的第一段，作為可信度的背書。

Claude Tag 目前以 beta 形式開放給 Enterprise 和 Team 客戶，運行在 Opus 4.8 上。管理者可以設定每個組織和頻道的 token 花費上限、查看 Claude 的所有操作日誌、以及限制 Claude 能存取的工具和資料。Claude Tag 會取代現有的 Claude in Slack app，既有管理者有 30 天可以選擇遷移。

Claude Tag 的產品頁面說這「只是開始」，目標是讓 Claude 出現在團隊工作的更多地方。目前是 Slack，接下來呢？Teams？Discord？Email？一個會在你不注意的時候讀你訊息、幫你排工作、主動推播「你該知道的事」的 AI。Anthropic 把這稱為協作，但公告中沒有討論這個產品和監控工具的界線在哪裡。

## 🧨 Claude Code 的隱形炸彈：安全閘門被偷偷改成倒數計時器

開發者 Olaf Alders 今天發表了一篇對 Claude Code v2.1.211 原始碼的逆向分析，挖出一個沒有出現在任何 changelog 中的行為變更：`AskUserQuestion`——這個被 hooks 和 rules 用做安全閘門的機制——從「阻塞等待使用者回應」變成了「60 秒倒數計時後自動放行」。

這不是無關緊要的 UI 改動。`AskUserQuestion` 在 Claude Code 的生態系裡被當作顯式的安全中止點：hooks 依賴它來阻擋危險操作（「你真的要對 production 執行這個指令嗎？」），rules 用它來要求人工確認。把一個阻塞閘門變成一個附帶 60 秒倒數的對話框，等於把安全機制的前提——「使用者必須明確同意」——悄悄置換成「使用者沒說不要就是同意」。

讓這件事更嚴重的兩個疊加因素：

**Claude Code 預設自動更新。** 一個沉默的行為變更 + 自動更新 = 你沒做任何事，安全閘門就在你背後失效了。Alders 在文章中特別點出這個組合的危險性：使用者可能在完全不知情的狀況下，用一個安全模型已經被改過的版本跑部署或 infra 操作。

**關閉自動更新不簡單。** Alders 整理了三種關掉自動更新的環境變數——`DISABLE_UPDATES`、`DISABLE_AUTOUPDATER`、`CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC`——但它們的優先序不一致、行為不同、而且關掉自動更新的同時也會凍結 plugin 更新（這點只在一份不相關的文件頁面提到過）。其中 `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` 是 presence-detected（只檢查變數是否存在，不檢查值），所以設成 `=0` 也會觸發。

Alders 的結論是：他不假設 Anthropic 有惡意，「事情總是會發生」。但「這種事會不會繼續發生」會決定 Anthropic 從這個 misfeature 中學到了什麼。

- 來源：[Olaf Alders](https://www.olafalders.com/2026/07/17/claude-code-anatomy-of-a-misfeature/)

## ⚖️ Apple 對 OpenAI 員工發出法律信函

《金融時報》報導，Apple 已對數十名 OpenAI 員工發出法律信函。由於 FT 文章在付費牆後方，具體內容——信函的性質（挖角限制？競業條款？智慧財產權？）、涉及哪些員工層級、以及雙方的法律立場——目前無法從公開來源確認。

這條新聞的時間點值得注意：Apple 和 OpenAI 在 2024 年 WWDC 宣布合作將 ChatGPT 整合進 Siri 和系統層級書寫工具，但據多家媒體後續報導，這項整合至今尚未在中國以外的市場大規模推出。Apple 同時也在開發自己的大型語言模型（代號 Ajax/Apple Intelligence 的一部分）。兩家公司在公開場合以合作夥伴互稱，但法律信函指向的是人才競爭的邏輯。
