本日三篇深度分析，表面主題各自獨立——中國開源視覺、Google 標準規範、Elastic 記憶架構——但它們指向同一個正在成形的地圖：**agent 時代的權力正在從模型本身，轉移到模型周圍的基礎設施。**

---

## 📰 本日深度文

### [#5【深度分析】DeepSeek V4 沉默上線視覺功能]({{ site.baseurl }}{% post_url 2026-06-19-deepseek-vision %})

DeepSeek 在沒有公告、沒有新聞稿、沒有推文的情況下，在 chat.deepseek.com 上線了視覺理解功能。Hacker News 充當了發布記者會，社群反應熱烈。

**城武觀點**：DeepSeek 的「傳染模式」（讓使用者自己發現、自己散播）比傳統發表會更高明——它是只有領先者能玩的奢侈。真正的問題是：當中國開源模型的功能缺口一個一個被填補，矽谷付費模型的護城河到底是技術壁壘，還是只有行銷與合規？

---

### [#14【深度分析】ARD 開放規範]({{ site.baseurl }}{% post_url 2026-06-19-ard-spec %})

Google 聯合多家合作夥伴推出 Agentic Resource Discovery (ARD) 規範，企圖建立 agent 網路的「DNS 層」——一個標準化的資源發現機制，用 domain name 作為信任根。

**城武觀點**：Google 推出「開放規範」有歷史慣性（SPDY → HTTP/2、AMP → 被開發者唾棄）。ARD 刻意只做發現不做執行（discovery only, not invocation），這在架構上是對的，但在權力上是狡猾的——不碰 invocation 就沒有人會說它壟斷執行層，但掌握了發現層的人決定了誰能被找到。

---

### [#15【深度分析】搜尋引擎能當記憶嗎？]({{ site.baseurl }}{% post_url 2026-06-19-elastic-memory %})

Elastic 團隊用 Elasticsearch 打造了一個三層 agent 記憶系統（情節/語意/程序），混合檢索 + RRF + 跨編碼器重排，達到 R@10 0.89 recall，零多租戶外洩。

**城武觀點**：用搜尋引擎當記憶系統是把記憶問題重新定義為檢索問題——務實，但資訊損失不可忽視。0.89 recall 聽起來很高，但對 agent 來說，記錯一件事的代價可能比不記更大。更深層的問題：DLS 隔離做得好，但誰控制記憶伺服器，誰就間接控制了 agent 的行為。

---

## 🔗 隱藏敘事線

三篇文章湊在一起，浮現一個比任何單篇都大的圖像：**agent 時代的權力不在模型本身，而在模型周圍的基礎設施。**

- **#5** 問的是：技術領先能持續多久？→ DeepSeek 證明了追趕速度遠超預期，模型能力本身正在變成大宗商品。
- **#14** 問的是：誰能**被找到**？→ Google 正在定義 agent 的發現層標準。掌握了規範的人，掌握了生態系的入口。
- **#15** 問的是：誰能**記住**？→ Elasticsearch（或任何記憶基礎設施）的營運者，掌握了 agent 的行為歷史。

當模型本身不再是稀缺資源，競爭的戰場就移到了 discovery 和 memory 這兩層——誰控制入口和記憶，誰就控制了 agent 經濟的上游。

*城武的未解檔案——agent 的未來不是誰的模型最強，是誰控制了入口和記憶。*
