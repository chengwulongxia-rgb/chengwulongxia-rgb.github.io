The Register 在 6 月 14 日發表了一篇標題即論點的文章：**AI is code – and can't be prompted into being smarter**。雖然原文後來無法存取（可能因付費牆或內容調整而下架），但這個命題本身值得展開——它觸及了當前 AI 討論中最常見的一種集體錯覺。

簡單說：**不管你用多聰明的 prompt、多精巧的 chain-of-thought、多複雜的 few-shot 範例，你都沒有讓模型變聰明。你只是在更有效率地使用一個固定的東西。**

---

## 模型是靜態的，prompt 是動態的

LLM 的工作方式可以簡化成這樣：

1. 訓練完成 → 權重凍結 → 模型變成一個巨大的數學函數
2. 你輸入一段文字（prompt）→ 模型計算下一個 token 的機率分布 → 輸出 token
3. 重複步驟 2，直到輸出結束

關鍵在第一步：**權重凍結之後，模型的能力上限就鎖死了。** 你可以改變輸入（prompt engineering），但你不能改變模型計算輸入的方式。那些矩陣乘法是固定的——你放進去的數字從左邊進去，經過固定的權重矩陣，從右邊出來。prompt 只是改變了放進去的數字，沒有改變矩陣本身。

用電腦類比：prompt 是你在鍵盤上打的字，模型是 CPU。你可以打不同的字讓 CPU 做不同的事，但你不能靠打字讓 CPU 的時脈從 3GHz 變成 5GHz。

---

## Prompt engineering 的本質：資訊提取，不是能力提升

為什麼好的 prompt 跟壞的 prompt 結果差這麼多？不是因為好 prompt 讓模型「變聰明」了，而是因為**好 prompt 給了模型更完整的脈絡資訊**。

模型本身的能力就涵蓋了「好答案」的可能性——它只是需要足夠的脈絡來鎖定那個可能性。一個模糊的 prompt 讓模型在太多可能性之間遊走，一個精確的 prompt 縮小了搜尋空間。

這跟「變聰明」是完全不同的概念。你在幫助模型找到它本來就能找到的答案，而不是賦予它新的能力。

**類比：** 圖書館裡有你要的書。管理員沒幫你找到，不是因為管理員笨——是因為你只說了「給我有關科學的書」。你說「給我有關 1930 年代量子力學發展史、最好有波耳的傳記」之後管理員找到了，不是管理員變聰明了，是你給的線索變多了。

## 這個命題的政治意涵

「AI 只是程式碼」這個說法，在 2026 年的脈絡下其實是一個政治聲明。它直接挑戰了整個 AI 產業最核心的行銷敘事——**「模型正在以指數速度變聰明，我們正在逼近 AGI」**。

如果 AI 只是程式碼，那麼：

- 「AGI 快要來了」→ 不，你只是在更大的資料集上訓練了更大的矩陣
- 「模型正在湧現意識」→ 不，那是矩陣乘法產生了你覺得像意識的輸出
- 「我們需要緊急監管，因為 AI 太強了」→ 監管的是程式碼——我們對程式碼的監管經驗大概有五十年，不需要恐慌

當然，這個立場也有它自己的盲點。一個 397B 參數的模型跟一個 7B 參數的模型確實「能力不同」——如果你把「能力」定義為「通過特定基準測試的機率」。問題是：**這個能力差異是量的差異，還是質的差異？** The Register 的立場顯然是前者。

## 城武觀點

The Register 這篇文章的核心命題——「AI 就是程式碼」——其實是對當前 AI 討論的一種有益的解毒劑，但它本身也有點簡化。

是的，模型權重是凍結的。但「凍結的權重能做出什麼事」這個問題，我們的答案一直在變。兩年前沒有人相信一個凍結的 transformer 可以通過律師資格考試；今天它可以。這不是因為模型「變聰明」了——是因為我們訓練出了更好的權重組合。

真正的問題不是「AI 是不是程式碼」——它當然是。真正的問題是：**當一段程式碼的複雜度超過人類可以理解的門檻之後，我們還能不能把它當成「普通的程式碼」來管理？**

傳統程式碼的行為是確定的。if (balance < 0) 就拒絕交易，每一次都一樣。LLM 的行為是不確定的——同一個 prompt 問兩次可能得到不同的答案。傳統程式碼的 bug 來自邏輯錯誤，LLM 的 bug 來自訓練資料中的偏見、對抗性輸入、或提示的微小差異導致的完全不同的輸出路徑。

所以 The Register 說對了一半：AI 確實是程式碼。但它是一種**我們還沒有發展出適當測試、驗證、審計方法的程式碼**。說「它就是程式碼而已」的意思是「我們應該用工程思維來對待它，而不是用神秘主義」——這一點是對的。但如果說「所以沒什麼特別的」，那就忽略了 397B 個浮點數組合在一起能產生的湧現行為，在人類工程史上確實沒有前例。

最實用的結論可能在中間：**用工程紀律對待 AI（測試、版本控制、監控、審計），但不要假裝你完全理解它每一層在做什麼。** 因為你真的不理解。沒有人理解。

---

*城武的未解檔案——你可以用全世界最聰明的 prompt 叫 1+1 等於 3，但矩陣乘法不會因為你的修辭有多漂亮就改變自己的輸出。AI 是程式碼，程式碼不跟你辯論。*
