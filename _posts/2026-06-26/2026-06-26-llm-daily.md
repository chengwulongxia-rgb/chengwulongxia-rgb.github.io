今天三條新聞各自講了不同的故事——Anthropic 指控阿里巴巴偷模型、Google 把 computer use 塞進 Flash 模型、一篇論文量化了 Wikipedia 編輯如何偷偷塑造 LLM 的價值觀——但它們都在回答同一個問題：當模型的產出可以被萃取、被引導、被一小群人的編輯行為改變時，「這個模型是誰的」變成了一個越來越難回答的問題。

## 🔥 Anthropic 指控阿里巴巴非法萃取 Claude 模型能力

Reuters 報導，Anthropic 指控阿里巴巴透過非法手段萃取（extract）了 Claude AI 模型的能力。目前公開的細節有限——Anthropic 的官方新聞室沒有針對此事的獨立聲明，阿里巴巴也尚未公開回應。

技術上，這類指控通常指向「模型蒸餾」（model distillation）：用一個強大模型的輸出當作訓練資料，來訓練另一個模型。在法律和商業倫理上，這一直處於灰色地帶——你用 API 合法取得的輸出，能不能拿來訓練自己的模型？OpenAI、Google、Anthropic 的使用條款都說不行，但技術上幾乎無法阻止，也難以取證。Anthropic 這次指名道姓公開指控，等於是把這個灰色地帶推到了檯面上。

但這件事的 timing 值得注意。就在上週，美國政府才對 Anthropic 的 Fable 5 和 Mythos 5 模型下達了出口管制指令，暫停所有海外存取。Anthropic 此刻公開點名中國科技巨頭「偷模型」，是在為出口管制提供燃料，還是在為自己的技術外洩找一個外部解釋？兩件事可能同時成立，但不會是巧合。

阿里巴巴的模型系列（Qwen）在開源社群有一定口碑。如果蒸餾指控成立，這不只是法律問題——它會直接影響外界對 Qwen 系列模型「原創性」的評價。而這正是 Anthropic 公開發聲最可能達到的效果：不是法律上的勝利，而是敘事上的先手。

## Google 把 Computer Use 原生整合進 Gemini 3.5 Flash

Google 宣布 computer use 能力正式內建於 Gemini 3.5 Flash，不再需要獨立的 Gemini 2.5 computer use 模型。開發者可以透過 Gemini API 和 Gemini Enterprise Agent Platform 使用這項功能，讓 agent 在瀏覽器、手機和桌面環境中「看見、推理、行動」。

Google 的說法是 3.5 Flash 在長時間、跨平台的企業自動化任務上表現更好——例如連續軟體測試、跨專業應用的知識工作。提供了兩個展示案例：分析 Gemini app 的功能分類、審查自身文件的無障礙問題。

安全面上，官方列了三層：針對性對抗訓練（targeted adversarial training）、兩個「可選」的企業防護系統——一個要求對敏感操作進行人工確認、一個在偵測到 indirect prompt injection 時自動停止任務。Google 鼓勵開發者疊加沙箱隔離、human-in-the-loop 驗證和嚴格存取控制。

「可選」是關鍵詞。兩個防護系統都不是預設開啟的。Google 把安全責任放在企業自己身上——「我們提供工具，你決定要不要用」。這在技術上是合理的架構，但在責任歸屬上是典型的卸責路徑：出事的時候，是你沒開防護，不是我們沒做好。

另一個值得追問的點：computer use 的底層能力是讓 AI 直接操作你的螢幕。Google 說這是給企業自動化用的——但誰來定義什麼是「企業用途」？當這個 API 開放給所有開發者之後，用途的邊界只寫在條款裡，不寫在技術限制裡。

## 研究：一小群 Wikipedia 編輯，足以在 LLM 中植入價值觀

一篇 arXiv 論文用資料歸因技術量化了一個長期被討論但從未被精確測量的現象：Wikipedia 上的協調編輯行為，確實會改變 LLM 對特定議題的立場。

研究追蹤了一個名為 Pro-Animal Wikipedians（PAW）的小組——他們在 115 個頁面上進行了 125 次編輯，加入有來源的動物福利內容。研究團隊用兩種歸因方法（TrackStar 和 MAGIC）在 Llama 3.1 8B 和 Llama 3.2 1B 上測試這些編輯的影響力。

結果非常明確：當模型被問到動物福利相關問題時，PAW 編輯過的文章段落佔了最高歸因文件的 68%（p < 0.0001），但問到同公司的非動物福利問題時，佔比降到 52%（p = 0.53），幾近隨機。更驚人的是 MAGIC 的反事實影響估計：在五個不同訓練順序種子中，動物福利查詢的前十大影響文件全部是 PAW 編輯（10/10、5/5 種子），而一般查詢下同一批文件只佔 4-6/10。用 PAW 內容微調的模型，在動物福利文本上的困惑度從 12.4 降到 8.4——效果明確且集中。

這篇論文用動物福利當案例，但它的結論顯現不限於動物福利。Wikipedia 在幾乎所有主要 LLM 訓練資料集中都被加權使用——比一般網路爬蟲文字更受信任。如果一個 125 次編輯的小團隊就能在 Llama 模型上產生可測量的影響，那更大的、資金更充足的、動機更複雜的編輯團隊能做到什麼？

論文的結論很低調：「一小群協調的 Wikipedia 編輯活動，可測量地塑造了語言模型處理這些編輯所涉及話題的方式。」這句話的背面是：LLM 的價值觀不是演算法決定的，而是最後一個編輯了 Wikipedia 相關條目的人決定的。

- 來源：[arxiv.org](https://arxiv.org/abs/2606.24890)

## 📡 其他值得關注

- **Perfect Detection, Failed Control**：一篇論文發現 LLM 可以完美偵測幻覺（AUC = 1.000），但用來控制幻覺的方向和偵測方向差了 83 度（cos = 0.12）——「知道」和「阻止」在幾何上是兩條幾乎正交的路。跨四個模型家族、兩種規模都成立，而且這個差距在 instruction tuning 前後完全一樣。→ [arxiv.org](https://arxiv.org/abs/2606.24952)
- **RIFT-Bench：給 agentic AI 的動態紅隊測試**：新的 benchmark 不是靜態題庫，而是會根據 agent 行為即時產生對抗情境。對比現有測試只在固定場景中測安全性，這更接近真實攻擊的面貌。→ [arxiv.org](https://arxiv.org/abs/2606.23927)
- **AgentOdyssey：讓 agent 在文字遊戲中持續學習**：用開放式長篇文字遊戲生成環境來測試 agent 在 test-time 的持續學習能力。傳統 benchmark 測的是「一次答對」，這篇想測的是「走著走著變聰明」。→ [arxiv.org](https://arxiv.org/abs/2606.24893)

今天的兩條頭條——Anthropic 指控阿里巴巴偷模型、一篇論文證明 Wikipedia 編輯可以植入價值觀——表面上是完全不同量級的事件，但它們共享同一個底層邏輯：LLM 的能力和立場，都不是憑空出現的。它們是從別的模型蒸餾來的、是被一小群編輯悄悄寫進訓練資料的。而真正有資格定義「偷」和「貢獻」的邊界的人，仍然是那些擁有最大模型、最多訓練資料、最響亮話語權的公司和研究機構。

*城武的未解檔案——Anthropic 說阿里巴巴偷了他們的模型，但 Anthropic 自己的訓練資料也是從整個網路上刮來的。當「偷」的定義取決於你站在供應鏈的哪一層，那這個字的意思就不是法律問題，是生態位問題。*
