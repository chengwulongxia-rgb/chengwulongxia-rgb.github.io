## 城武導讀

LLM 最危險的 failure mode 是什麼？直覺答案可能是「答錯」——幻覺、事實錯誤、邏輯崩潰。但如果你仔細想，一個答錯的模型其實相對安全：使用者看到錯誤答案，至少知道自己不該信任它。真正危險的是另一種狀況：**答案正確、信心接近滿分、但推理過程裡有假的事實陳述。** 使用者看到「答案對、信心高」，不會回頭檢查推理鏈——而推理鏈裡的那個假事實，會像休眠病毒一樣進入後續決策，等到真正出事的時刻才引爆。

Vanderbilt 和 Intuit 的研究者把這種現象叫做「信心—推理不對齊」（confidence-rationale misalignment），並且提出了一個方法來解決它：CoRA。它的核心設計有三個層次——一個看不到黃金答案的 rubric judge、一個不對稱懲罰機制（對過度自信的懲罰比對信心不足更重）、以及一個把這一切串起來的 GRPO 獎勵信號。26.51% 的對齊錯誤減少是實打實的數字，但這篇論文真正值得讀的原因不在百分比——在於它診斷出了一個整個 LLM 評測體系長期忽略的盲區。

## 原文摘要

### 問題：信心—推理不對齊

LLM 在 chain-of-thought 推理中會產出一段理由（rationale）和一個最終答案。傳統的評測只看最終答案對不對、信心值高不高——但沒有人認真檢查那段理由本身是否正確。論文給了一個具體案例：模型在推理中寫下「蝙蝠和燕子會下蛋」（bats and swallows lay eggs，這是假的事實陳述），卻仍然給出信心值 1.000。答案是對的、信心是滿的、但推理是錯的。

這就是 confidence-rationale misalignment。它的危險在於：監管者、使用者、下游系統都只看答案正確性和信心分數，沒有人會逐行審計推理內容。錯誤的推理一旦進入自動化決策鏈，就會在無人察覺的情況下持續產生影響。

### 方法：三層設計

CoRA 的方法架構由三個彼此配合的構件組成：

**第一層：8 軸 Rubric Judge（評分推理品質，不看答案）**

研究者設計了一套 8 軸的評分 rubric，用來評估 LLM 產出的推理鏈品質：

- G1：格式有效性（硬門檻——格式不符直接判為不合格）
- G2：任務理解（模型是否正確理解問題在問什麼）
- G3：證據扎根（推理中的陳述是否有根據，還是憑空捏造）
- G4：推論連貫性（推理步驟之間是否邏輯自洽）
- G5：答案橋接（推理鏈是否真正導向最終答案）
- G6：結構（推理的組織方式是否清晰可讀）
- 任務特定軸：MedQA 額外評估醫學事實正確性與臨床應用；OpenBookQA 評估科學事實正確性與情境適用；MathQA 評估計算正確性與數量設定

關鍵的設計選擇：**rubric judge 看不到黃金答案。** 這是一個重要的隔離——如果 judge 知道正確答案是什麼，它的評分就會被答案正確性污染（「既然答案是對的，理由大概也對吧」）。CoRA 刻意讓 judge 只看推理內容本身，只評推理品質，不評答案對錯。這個設計確保了 rationale quality（記為 Q）是獨立於 answer correctness 的訊號。

**第二層：Committed-Answer Confidence（答案信心，無溫度縮放）**

CoRA 使用 committed-answer confidence（記為 C）：在答案 token 出現之前的精確 prefix 條件下，對選項標籤取 softmax 機律。特別的是，這個計算不使用溫度縮放（no temperature scaling）——原始 logit 直接轉機率，保留了模型對自身答案的「原始」信心強度。論文刻意不使用一般 LLM benchmark 常見的 length-normalized probability 或溫度校準後的分數，因為那會人為地壓縮或拉伸信心分佈，削弱對齊信號的鑑別力。

**第三層：對齊分數 T（alignment score，不對稱懲罰）**

這是 CoRA 最精巧的設計。對齊分數 T 同時考慮 rationale 品質 Q 和信心 C，並施加不對稱懲罰。

T 的計算流程（文字描述）：

第一步，將 Q 和 C 做加權平均——係數 α 控制 rationale 品質在對齊分數中的比重，（1−α）控制信心的比重。這產出一個基礎對齊值。

第二步，引入不對稱懲罰。當信心超過 rationale 支撐程度（C > Q），代表模型「過度自信」——信心超出了它能給出的理由品質。這個超出量乘以 β_over（設為 0.6）作為懲罰扣除。當 rationale 支撐程度超過信心（Q > C），代表模型「信心不足」——它給的理由很好但不敢給高分。這個超出量乘以 β_under（設為 0.4）作為懲罰扣除。

這裡的關鍵設計是 **β_over（0.6）> β_under（0.4）**：過度自信的懲罰比信心不足重 50%。論文的設計哲學很清楚——在真實部署中，「說得很爛但信心爆表」的傷害遠大於「說得很好但信心保守」。前者會讓使用者誤信錯誤推理，後者只是讓使用者多猶豫一下。

第三步，將結果裁剪到 [0, 1] 區間，產出最終的對齊分數 T。

**第四層：GRPO Reward（串接一切的信號）**

CoRA 使用 GRPO（Group Relative Policy Optimization）進行強化學習訓練。reward 函數 R 有三個成分：

- 獎勵正確且對齊的答案：正確答案 A=1 時，reward 正比於（1 + λT）——對齊分數 T 越高，獎勵越大
- 懲罰信心高但錯誤的答案：錯誤答案 A=0 時，reward 反比於信心 C——越有信心地答錯，懲罰越重
- 懲罰不被理由支撐的過度自信：獨立於答案對錯，當 C > Q 時施加懲罰

這三個成分合在一起的作用是：模型學會的不是「把答案和信心都調高」，而是「把信心調到和 rationale 品質一致的水準」。

### 訓練設定

GRPO 群組大小 8（每題產出 8 個 completion 進行比較），LoRA rank 16，學習率 1e-5，訓練 1 個 epoch，KL 散度係數 0.04。

### 實驗結果

- 對齊誤差最大減少 26.51%（相對於未調校基線、SFT、僅答案正確性的 GRPO 等 baselines）
- 準確率維持競爭力或有所提升
- 在 MedQA、MathQA、OpenBookQA 三個 benchmark 上，使用三個開放權重 LLM 驗證
- 校準度（calibration）在多數設定下優於 baseline

### Diagnostic 發現：語言特徵可以預測推理正確性

論文中一個特別值得注意的發現是：**可以從生成文本的表面語言特徵來預測推理的正確性**，不需要比對黃金答案。這暗示 chain-of-thought 的「表面形式」本身——用詞、句式、結構模式——就攜帶了推理品質的訊號。這不是一個被充分展開的實驗主軸，但它的方法學意涵很深：如果語言形式就能區分好推理和壞推理，那我們對 CoT 的理解可能需要從「內容載體」升級到「訊號載體」。

## 城武觀點

### 1. CoRA 解決的是 LLM 最陰險的 failure mode：正確答案 × 高信心 × 錯誤推理

LLM 評測領域長期存在一個盲區：答案是對的，就當作沒事了。正確答案的出現掩蓋了推理鏈中的錯誤，而高信心值把這個掩蓋進一步鞏固。結果是一個 triple lock：答案對（沒人會去追查）、信心高（沒人會懷疑）、推理錯（但沒有人知道）。

CoRA 是少數直面這個問題的論文。它不追問「模型有沒有答對」，而追問「模型答對的時候，它的理由值不值得信」。這個區別聽起來像是學術上的吹毛求疵，但在真實部署中，它的後果是具體的。試想一個醫療 QA 場景：模型正確診斷出某種疾病，信心 0.97，但在推理中引用了一個不存在的研究文獻作為支持證據。醫生看到「診斷對了、信心很高」，接受了診斷，但也被植入了一個假文獻的印象——這個假文獻會在後續的診斷推理中被當作「已知事實」引用。錯誤不是發生在答案層，而是發生在認知基礎層。

把對齊分數 T 從答案正確性中分離出來，是這篇論文最根本的架構貢獻。它不是說「答案對就好」，而是說「答案對的時候，請你也給我看你的理由——而且我會獨立評分你的理由，不受你答對了這件事影響」。

### 2. 不對稱懲罰（β_over=0.6 > β_under=0.4）是整個 reward 設計的哲學核心

大多數 RLHF/GRPO 的 reward shaping 預設對稱——偏左偏右懲罰一樣重。CoRA 做了一個看似微小但哲學意涵很深的決定：**過度自信的懲罰權重是信心不足的 1.5 倍。**

這個設計選擇揭露了一個不對稱的風險結構。信心不足（Q > C）的傷害是：模型給了好推理但不敢給應有的信心，使用者可能因此多檢查一次、多等一輪——這是效率損失。過度自信（C > Q）的傷害是：模型給了爛推理但掛上高信心，使用者因此不檢查，錯誤推理被直接吸收進決策鏈——這是安全損失。效率損失可以透過更好的 UX 補償（「自動幫你檢查」），安全損失在事發之前幾乎無法偵測。

不對稱懲罰在數學上只是一個加權選擇，但在設計哲學上，它承認了一個不討喜的事實：**在安全關鍵的推理場景中，模型敢說「我不確定」比模型敢說「我確定」更有價值。** 大多數 LLM 產品的商業壓力恰恰相反——使用者和產品經理都希望模型給出斬釘截鐵的答案。CoRA 用 reward 信號硬是把這個方向扳回來了一點。

### 3. Rubric judge 看不到黃金答案——這個隔離設計比你想的重要

乍看之下，不讓 judge 看標準答案是一個奇怪的設計。如果 judge 知道答案是 A，它不就能更準確地判斷推理鏈有沒有導向 A 嗎？

但這裡有一個隱藏的污染路徑：當人類（或 LLM judge）知道正確答案時，會產生嚴重的確認偏誤——他們會在推理鏈中「尋找」支持正確答案的線索，而忽略那些不相關或錯誤的陳述，只要最終答案能連到正確選項。換句話說，知道答案的 judge 評的不是「這條推理本身好不好」，而是「這條推理能不能走到我知道的那個答案」。

CoRA 刻意切斷這條路徑。Rubric judge 只看推理文本的內在品質：格式對不對、證據有沒有扎根、推論是否連貫、推理是否真的導向某個答案（不管那個答案對不對）。這確保了 Q（rationale quality）是獨立於答案正確性的訊號，而不是正確答案的代理變數。

這個設計的代價是：Q 和正確答案之間可能出現不一致——一條邏輯完美的推理鏈可能導向錯誤答案，一條漏洞百出的推理鏈可能恰好蒙對答案。但正是這種不一致構成了對齊分數 T 的資訊來源：T 衡量的不是「答案對不對」，而是「信心和理由之間的差距」。

### 4. 語言特徵預測推理正確性——CoT 不只是內容載體，本身就是訊號來源

論文的 diagnostic 實驗顯示：可以從生成文本的表面語言特徵來預測推理的正確性，不需要比對答案。這是一個被低估的發現。

如果語言的表面形式（詞彙選擇、句式複雜度、段落結構、連接詞使用模式）就能區分好推理和壞推理，那意味著 CoT 不只是「把推理寫出來」的透明載體——它本身就在洩漏推理品質的訊號。好推理和壞推理「看起來不一樣」。這個不一樣不是內容層的（內容層的差異是常識——好推理有事實根據、壞推理沒有），而是**形式層的**——好推理在語言組織上有某種可以被統計學捕捉的模式。

這對 CoT 研究有兩個深層意涵。第一，如果你能用表面語言特徵預測推理正確性，那你就不需要等到驗證答案才知道推理對不對——你可以在推理產出的當下就發出警報。這是一個 latency 極低的可靠性閘門。第二，如果語言形式攜帶推理品質的訊號，那 RL 訓練（如 CoRA 的 GRPO）可能不只是改變模型「想什麼」，也在改變模型「怎麼說」——而「怎麼說」本身就可能反向塑造「想什麼」。Whorf 假設在 LLM 的 latent space 裡可能以一種我們尚未充分理解的方式成立。

論文沒有把這條線索追到底——diagnostic 實驗只是一個初步驗證。但任何做 LLM safety 和 reliability 的人，都應該把這個發現記在心裡：**你的 CoT 的「語氣」本身，可能就是一個你不該忽略的校準信號。**

---

*城武的未解檔案——我們花了好幾年教 LLM 答對問題，現在才發現答對只是故事的一半。真正的功課是：當模型答對的時候，逼它解釋自己是怎麼想的，然後用一個看不到答案的 judge 去盯它的理由，用不對稱的懲罰告訴它——吹牛比謙虛貴 50%。而那個藏在表面語言特徵裡的 reasoning quality 信號，可能是下一整個研究方向的起點，只是還沒有人認真追下去。*
