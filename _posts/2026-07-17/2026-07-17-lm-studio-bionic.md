如果你是用開放模型的開發者，LM Studio Bionic 聽起來像是為你量身打造的答案——一個能在本地跑、能接雲端、能寫程式、能處理文件的 AI agent。但當你仔細讀完整篇公告，你會發現一個藏在「開放」旗幟底下的問題：這東西到底是為開放生態服務，還是為 LM Studio 自己服務？

## 原文深度翻譯

LM Studio 推出了他們自稱「演化史上最大的一步」——LM Studio Bionic，一個專為開放模型打造的 AI agent。Bionic 的定位是「用開放模型完成真正的工作」，涵蓋程式開發、研究、以及文件與檔案的複雜處理。你可以用本地模型，也可以切換到雲端的開源模型來處理更重的任務，同時保持對隱私和 AI 費用的控制。

LM Studio 對所有 Bionic 使用者做出了一項承諾：Zero Data Retention（零資料留存），並且絕不用你的資料來訓練模型。

Bionic 的核心功能涵蓋四個面向：

**語音輸入與本地轉錄。** Bionic 內建了語音鍵盤，使用 Mistral AI 的 Voxtral 模型進行完全離線的語音轉錄。Voxtral 是一個高效能的多語言即時轉錄模型，所有處理都在裝置端完成，語音資料不需上傳雲端。使用者可以在任何 app 中啟動語音鍵盤，Bionic 會從游標所在位置開始轉錄——講出你的想法、提示詞、編輯指令，全程離線完成。

**程式開發。** Bionic 支援廣泛的程式開發需求，主打「不放棄隱私和控制」。它可以檢視本地程式碼庫、解釋不熟悉的程式碼、幫助你進行修改。建立一個 Code 專案並指向本地資料夾後，你可以叫 Bionic 調查、編輯、除錯，並即時檢視它的工作成果。內嵌的 diff 檢視讓每行程式碼變更都一目了然；agentic 程式碼搜尋則讓 Bionic 能快速找到相關檔案、追蹤行為、解釋陌生程式碼。Bionic 相容於 GLM 5.2 和 Kimi K2.7 Code 等強大的開放模型，讓你在控制成本的同時完成更多工作。

**文件、簡報與試算表。** Bionic 也定位為通用生產力和深度知識工作的工具。你可以丟文件給它處理，或要求它從零生成新文件、簡報、試算表。在 Work 專案中，Bionic 於沙箱環境內處理文件，確保電腦上的其他檔案安全無虞。它能整理本地目錄、編輯檔案、摘要素材，並透過原生網頁搜尋引入外部脈絡。自動檢查點讓你可以安全地檢視或回溯變更，應用內預覽則將素材和工作流程集中在一個地方。LM Studio 表示正在持續增加更多檔案類型的預覽支援。

**原生本地執行。** Bionic 內部可直接下載並執行本地模型，使用 LM Studio runtime 驅動。從簡單對話到進階 agentic 任務，都可以用本地模型完成。

**雲端推論與預設零資料留存。** Bionic 也支援最新前沿開放模型的雲端推論，透過 LM Studio Secure Cloud 執行。隨著前沿開源模型在程式開發、推理、工具呼叫和長上下文任務上不斷進步，Bionic 讓使用者可以在 LM Studio Secure Cloud 中試用這些模型。使用雲端模型時，請求是瞬態處理的，請求完成後不會被保留。

**入門方式。** Bionic 是一個全新的獨立 app，與原本的 LM Studio 分開。需要進階低階設定的使用者可以繼續使用 LM Studio 搭配 Bionic。使用雲端模型需要建立 LM Studio 帳號來設定計費。連接專案、選擇模型，就可以開始跟 Bionic agent 一起工作。

LM Studio 表示會隨著開放模型能力的成長以及使用者實際專案中的回饋，持續改善體驗。

## 城武觀點

**「給開放模型的 AI agent」——這句話本身就是矛盾的。**

如果模型真的開放，為什麼需要一個專屬的 agent 平台？真正開放的生態應該是：agent 能換、模型能換、雲端服務能換。你可以在 Cursor 裡用 Llama、在 Terminal 裡用 Qwen、在自架的 vLLM 上跑 Mixtral——agent 是殼，模型是心，服務是腿，三者各自獨力。但 LM Studio Bionic 把三者綑綁在一起：你用它的 agent、跑它支援的模型（雖然是開源的）、透過它的 Secure Cloud 做雲端推論。這不是開放，這是在開放模型的屍體上蓋自己的城堡。LM Studio 說「我們是給開放模型的 agent」，但更精確的描述是「我們是讓開放模型只能在我們平台上當 agent 的 agent」。換句話說，LM Studio 把「開放」從一種生態屬性降級成一個產品差異化標籤。真正的威脅不是閉源模型——閉源模型從不假裝開放。真正的威脅是把開放當作行銷術語的封閉平台，因為它會讓你放下戒心。

**「Zero Data Retention」——沒有審計的財務報表。**

LM Studio 的隱私成諾聽起來很乾淨：零資料留存、絕不訓練你的資料。但問題是：誰來驗證？在沒有第三方稽核、沒有開放原始碼的伺服器端、沒有可獨立驗證的資料處理記錄的情況下，「我們不保留你的資料」這句話，跟一家公司說「我們財報沒問題，不用審計」沒有本質區別。信了是你的事。更深的問題是：這類承諾正在成為一種新的「隱私清洗」——企業用一個聽起來很絕對的承諾換取你的信任，但把驗證的責任完全丟給使用者。你無法驗證，所以你只能選擇相信或離開。而大多數人會選擇相信，因為「Zero Data Retention」這幾個字太乾淨了，乾淨到你不忍心懷疑它。

*城武的未解檔案——開放模型的未來不應該長成另一個有圍牆的花園，不管牆上貼了多少張「我們很開放」的貼紙。*
