## 合作公告全文翻譯

### TCS 成為 Anthropic 合作夥伴網路首家全球系統整合商

2026 年 6 月，Anthropic 宣布與塔塔顧問服務（Tata Consultancy Services，TCS）達成戰略合作。TCS 是全球最大的 IT 服務公司之一，在 56 個國家擁有業務，員工人數超過六十萬。這項合作的核心內容是：TCS 將把 Claude 部署給其全球五萬名員工，涵蓋工程、財務、法務、行銷、業務等部門；同時，TCS 將以 Claude 為基礎，為金融服務、醫療保健、公共部門等受監管產業建構專屬解決方案。

TCS 是首家加入 Anthropic 合作夥伴網路（Claude Partner Network）的全球系統整合商。這意味著 TCS 不只是轉售 API，而是獲得 Anthropic 的官方認證，成為 Claude 在企業市場的正式部署夥伴。

### 「Customer Zero」：五萬人先當白老鼠

TCS 執行長 K. Krithivasan 在公告中特別強調了一個概念：**TCS 將以「customer zero」的角色，先在自己內部全面採用 Claude，再將驗證過的部署模式帶給客戶。** 這個策略的核心邏輯是——與其花兩年跟客戶做 PoC（概念驗證），不如先在自己的五萬名員工身上跑過一輪。工程團隊用 Claude Code 加速開發、法務用 Claude 審閱合約、行銷用 Claude 產生素材——這些內部使用的經驗、痛點、最佳實踐，會直接轉化為對客戶的部署方案。

塔塔集團董事長 N. Chandrasekaran 也為這項合作背書，強調塔塔集團在受監管產業的深厚經驗，將確保 Claude 的部署符合最嚴格的安全與合規要求。

### 產業應用：從理賠到程式碼

合作的具體落地場景包括：

- **Diligenta**：TCS 在英國的子公司，專門管理人壽與退休金保單，服務超過 2,200 萬名保單持有人。Diligenta 將使用 Claude 改善客戶體驗——從理賠查詢到保單變更，這些過去仰賴大量人工處理的流程，將逐步由 Claude 驅動的系統輔助。
- **銀行與金融產品團隊**：TCS 的銀行團隊正在使用 **Claude Code** 提升軟體工程生產力。更具體地說，TCS 工程團隊正在為 Claude Code 開發可重複使用的技能與外掛，針對特定業務場景——例如理賠裁定（claims adjudication）、貸款諮詢（lending advisory）——進行客製化。
- **保險公司**：Claude 將被應用於理賠處理流程，協助保險公司在合規框架內加速理賠決策。
- **銀行貸款諮詢**：透過 Claude 驅動的系統，提供貸款顧問即時的合規指引與風險評估。

### TCS iON：七千五百萬次評測的基礎設施，加上 Claude 認證

TCS iON 是 TCS 旗下的教育科技平台，每年在印度 1,500 個城市執行超過 **7,500 萬次評量**，涵蓋學校考試、技能認證、政府招聘測驗等。根據合作公告，TCS iON 將推出以 Claude 為基礎的培訓與認證課程——換句話說，每年觸及七千五百萬人次的評測基礎設施，將開始產出「Claude 認證」的專業人員。

### 高層語錄

Dario Amodei 在公告中明確指出：「**受監管產業需要他們的工作成果具有高度準確性與可審計性。**」他進一步補充：「這項合作深化了我們對印度的承諾——**印度是 Anthropic 的第二大市場。** 印度企業正在大規模應用 AI，重新構想其流程與客戶體驗。」

Krithivasan 則強調：「企業 AI 的真正價值，來自於理解業務情境、協調複雜系統，以及應用深厚的 AI 工程人才。TCS 與 Anthropic 的這項合作，正是建立在我們對這些原則的共同承諾之上。」

## 城武觀點

### 1. 「Customer zero」不是行銷話術——是企業 AI 最務實的進入策略

「Customer zero」這個詞聽起來像顧問公司發明的另一個 buzzword，但如果你把它還原到企業 AI 落地的現實困境裡，會發現它是目前最務實的策略。

企業 AI 銷售有一個死結：客戶不敢大規模採用，因為沒有足夠的證據證明它在自己這個產業行得通；但沒有客戶大規模採用，就永遠不會有證據。傳統的做法是 PoC——花三個月、砸幾十萬美金，在一個隔離的 sandbox 環境裡跑一個不痛不癢的用例，然後產出一份「成功導入」的簡報。但 PoC 跟生產環境的距離，大概跟游泳池跟大西洋的距離差不多。

TCS 的「customer zero」模式把這個死結一刀切開。與其在客戶端做 PoC，不如在自己的五萬名員工身上做 **full-scale production deployment**——不是隔離環境，是真實的日常工作。法務真的在用 Claude 審合約，不是假裝在審。工程師真的在用 Claude Code 寫 production code，不是玩具專案。行銷真的在用 Claude 產生 campaign 素材，不是 demo 用的假句子。

這裡面的策略縱深比表面看起來更深。第一，TCS 自己就是一個跨 56 國的巨型企業——它的內部部署場景本身就跟它的目標客戶高度重疊。一個在 TCS 內部驗證過的金融合規 Claude 部署方案，搬到客戶的銀行時，需要調整的變數遠少於從零開始。第二，TCS 的員工同時也是它客戶的顧問——當這些顧問自己每天在用 Claude，他們對客戶的銷售就不再是「這個產品理論上可以幫你」，而是「我每天都這樣用，我告訴你哪裡會痛、哪裡要小心、哪裡最有效」。這種銷售轉換率，跟拿著原廠投影片照念的經銷商，不在同一個量級。

但這個模式的真正威力不在銷售——在於**它把「部署風險」從客戶端轉移到了 TCS 自己身上**。傳統企業軟體銷售的隱含前提是：客戶承擔部署失敗的風險。PoC 失敗了，客戶浪費三個月跟幾十萬美金。但「customer zero」模式下，TCS 先在內部把會爆的雷全部踩過一輪，把解決方案打磨到能用的程度，才拿出去賣。客戶買到的不是一個「應該會動」的產品——是一個「在五萬人的生產環境裡已經跑過」的產品。

這不是 revolution，這是 common sense。但 common sense 在企業 AI 市場裡居然是一種競爭優勢——可見這個市場有多不成熟。

### 2. 受監管產業是 LLM 最難打入的市場——也是價值最高的市場

如果你畫一張 AI 落地的難度／價值矩陣，左上角（低難度、低價值）是客服摘要和行銷文案；右下角（高難度、高價值）是銀行核心系統、保險理賠、醫療診斷輔助、政府服務。過去三年，幾乎所有 LLM 應用都擠在左上角。右下角一直是禁區——不是因為技術做不到，而是因為這些產業對「可審計性」的要求，跟 LLM 的機率本質有結構性衝突。

銀行不能接受「這個貸款審核大概 87% 的時候是對的」。監管機關要求每一筆決策都必須可以追溯、可以解釋、可以被複審。但 LLM 的本質是一個機率分布——它給你最可能的輸出，不是唯一正確的輸出。這個落差不是靠「更好的模型」能解決的——GPT-5 再聰明，它的輸出仍然是機率性的，而監管機關要的是確定性。

Anthropic 的切入策略很聰明：它不打「模型最聰明」這張牌，**打的是「安全、可審計」這張牌**。Dario 在公告中的那句話——「受監管產業需要高度準確性與可審計性」——不是隨口說說。這是 Anthropic 在重新定義競爭維度。當其他 AI 公司還在比 benchmark 分數時，Anthropic 在比誰的模型能通過監理機關的稽核。

但這裡有一個不該被跳過的問題：**Claude 的「可審計性」具體是什麼？** Constitutional AI 是一套訓練方法，讓模型的輸出更符合特定的價值框架。但「符合 Anthropic 定義的安全標準」跟「符合印度儲備銀行或英國 FCA 的監管標準」是兩回事。前者是自我定義，後者是外部驗證。TCS 的價值正是在這裡：他們在這些監管框架下有幾十年的合規經驗——他們知道 RBI 要什麼、FCA 要什麼、FDA 要什麼。Anthropic 不需要從零開始學習如何跟這些監管機構對話——TCS 已經有現成的合規框架，Claude 要做的只是嵌入其中。

換句話說，這筆交易的本質是 **「借信」**：Anthropic 借 TCS 的產業信任與合規資產，TCS 借 Anthropic 的 AI 技術光環。雙方互補的深度，遠超過一般的經銷關係。

### 3. 「印度是第二大市場」——Dario 親口說出的地緣政治訊號

Dario Amodei 在公告中說了一句大多數人可能當成背景資訊就滑過去的話：「印度是 Anthropic 的第二大市場。印度企業正在大規模應用 AI，重新構想其流程與客戶體驗。」

這句話的重量，要放在兩個脈絡裡才能秤出來。

**第一個脈絡：美國的 AI 出口管制正在收緊。** Anthropic 的 Fable 5 / Mythos 5 剛被美國政府列入出口管制清單，禁止銷售給外國實體。雖然這次合作部署的是 Claude（一般版本），不是被管制的模型，但方向很清楚——當美國政府把 AI 技術當成戰略資產在管制時，矽谷 AI 公司的「母國市場」正在被政策力量縮小。在這種情況下，**「第二大市場」遲早要變成「第一大市場」**——不是因為印度成長得多快，而是因為美國市場被政策掐住了。

**第二個脈絡：印度不只是人口多，是全球 IT 勞動力的心臟。** TCS 雇用超過六十萬人，Infosys、Wipro、HCL 加起來又是幾十萬。全球的銀行核心系統、保險後台、電信營運——很大一部分的程式碼是印度工程師寫的、印度團隊維護的。當 Anthropic 說「印度是第二大市場」，它指的不是印度人在用 Claude 聊天——指的是 Claude 正在進入這些印度 IT 服務巨頭的日常工作流程，而這些巨頭的客戶遍布全球。

這才是真正的地緣政治棋盤重劃。過去三十年，矽谷的軟體公司透過印度 IT 服務商把產品賣到全世界——Oracle、SAP、Salesforce 都是這條路。但那些是軟體工具，換供應商的成本雖然高，不是不可能。AI 模型的生態鎖定——如果你把 Claude 嵌進了 TCS 為客戶建置的銀行核心系統，那個切換成本就不只是換 API endpoint 那麼簡單。它牽涉到重新訓練整批工程師、改寫所有 Claude Code 的外掛和技能、重新通過監管機關的合規審查。

Dario 的那句話與其說是市場數字，不如說是地緣政治宣言：**如果美國政府不讓我們在美國長大，我們就把根扎在印度。** 而且印度這塊土壤——全球最大的 IT 勞動力池、相對友善的監管環境、跟西方的語言與制度親近性——可能是 Anthropic 在美國以外能找到的最佳替代市場。

### 4. TCS iON 的七千五百萬次評測：這不是培訓，這是「Claude 認證專業人員」的生產線

整份公告中最容易被忽略、但長期影響可能最深的，是 TCS iON 的那一段。

TCS iON 每年在印度 1,500 個城市執行超過 7,500 萬次評量。這個數字的規模需要被正確理解：七千五百萬次——不是累積用戶數，是**每年**的評測次數。這是一個覆蓋全印度教育與技能認證體系的基礎設施。從學校期末考到政府招聘，從職業技能認證到企業內部升等測驗，TCS iON 是印度最大的人才評測平台之一。

現在，這個平台要加入 Claude 培訓與認證。

這不是「開幾堂線上課程」等級的動作。**這是在一個每年觸及七千五百萬人次的平台上，嵌入一個特定 AI 模型的認證軌道。** 想像一下：一個印度工程系畢業生，未來的履歷上不是只有「熟悉 LLM」，而是「TCS iON Claude 認證——理賠裁定專業級」、「TCS iON Claude 認證——貸款諮詢進階級」。這些認證會直接掛在 TCS iON 這個已經被印度企業和政府信任的評測品牌之下。

這裡的策略邏輯跟 Cisco 在 1990 年代做的事情一模一樣。Cisco 不是只賣路由器——它在全球建立了 CCNA/CCIE 認證體系，讓數十萬網路工程師的職業生涯圍繞著 Cisco 的技術棧建立起來。當一個企業要採購網路設備時，決策者想的不只是「哪家設備最好」，而是「我找得到多少 Cisco 認證的工程師來維護這套系統」。生態鎖定不是發生在採購端——**是發生在勞動力市場端。**

Anthropic 顯然看到了同樣的劇本。TCS 不只是幫 Claude 賣進銀行和保險公司——TCS 在幫 Claude 建立一個「認證專業人員」的 pipeline。當印度每年有成千上萬的工程師透過 TCS iON 取得 Claude 認證，這些工程師進入職場後，自然會把 Claude 帶進他們的雇主。他們是 Claude 的活廣告、活部署團隊、活生態節點。

而且這個 pipeline 的進入障礙高到很難複製。TCS iON 的七千五百萬次評測基礎設施不是一天建成的——那是跟印度各級政府、教育機構、企業 HR 部門長達十幾年的合作關係和信任積累。其他 AI 公司想要在印度建立同等規模的認證通路，不是沒有可能，但時間窗口正在關閉。**先佔據認證通路的人，會定義這個市場的「預設選項」。**

### 5. 把四件事放在一起看：Anthropic 的企業策略正在成形

如果你把「customer zero」、「受監管產業的可審計性定位」、「印度第二大市場」、「TCS iON 認證 pipeline」這四個命題疊在一起，一個清晰的策略圖像會浮現出來：

Anthropic 不打「模型最便宜」這張牌（那是開源模型的賽道），不打「模型最聰明」這張牌（那是 OpenAI 和 Google 在燒錢互毆的賽道），**打的是「模型最安全、部署最踏實、生態最難逃脫」這張牌。**

- 安全 → 進入受監管產業的門票（銀行、醫院、政府不敢用不安全的 AI）
- 部署踏實 → customer zero 模式降低客戶採用風險（五萬人先跑過，你才買）
- 生態難逃脫 → TCS iON 認證 + 印度勞動力鎖定（當你的工程師都是 Claude 認證的，換供應商等於換掉整個人力資源體系）

這三個要素構成了一個複合護城河——不是單靠技術領先（那種領先很容易被追上），而是靠安全聲譽、部署信任、勞動力生態的三重鎖定。當你的競爭對手還在想辦法讓自己的模型在某個 benchmark 上多拿兩分的時候，你以經在印度的每一所大學裡鋪好了認證管線。

這不是說 Anthropic 不重視模型研發。而是說，**Anthropic 認知到模型能力的邊際競爭價值正在遞減，而生態鎖定的邊際競爭價值正在遞增。** 它正在把資源從前者轉移到後者——用 TCS 這筆交易作為轉向的第一個大型錨點。

---

*城武的未解檔案——當一家 AI 公司不再吹噓自己的模型有多聰明，而是開始認真談「可審計性」、「customer zero」、「認證 pipeline」的時候，它不是變得謙虛了。它是發現了另一種贏的方式——不是跑得比所有人快，而是讓所有跑得比你快的人到了終點才發現，門鎖的鑰匙在你手上。*
